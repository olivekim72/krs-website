# 🌹 회원 스토리 기능 켜는 방법 (관리자용)

회원들이 **로그인 → 사진과 글을 올리면 모든 회원이 함께 보는** 스토리 게시판입니다.
이 기능은 무료 서비스인 **Supabase**(클라우드 데이터베이스)를 한 번만 연결하면 작동합니다.
코딩 없이, 아래 순서대로 따라 하시면 됩니다. (약 10분)

---

## 1단계 — Supabase 프로젝트 만들기

1. https://supabase.com 접속 → **Start your project** → 가입(깃허브/구글/이메일).
2. **New project** 클릭 → 프로젝트 이름(예: `krs-website`), 비밀번호 입력 → 생성.
3. 생성이 끝날 때까지 1~2분 기다립니다.

## 2단계 — 데이터베이스 준비 (복사·붙여넣기)

1. 왼쪽 메뉴에서 **SQL Editor** 클릭 → **New query**.
2. 이 폴더의 **`supabase/schema.sql`** 파일을 열어 내용을 전부 복사합니다.
3. SQL Editor 에 붙여넣고 오른쪽 아래 **Run** 버튼 클릭.
4. "Success" 가 나오면 테이블·보안·이미지 저장소가 모두 만들어진 것입니다.

## 3단계 — 이메일 인증 끄기 (테스트 편하게)

1. 왼쪽 메뉴 **Authentication** → **Providers** → **Email** 클릭.
2. **Confirm email** 옵션을 **끕니다(Off)** → Save.
   - (이 옵션을 켜두면 가입 시 이메일의 인증 링크를 눌러야 로그인됩니다.)

## 4단계 — 연결 정보 입력하기

1. 왼쪽 메뉴 **Project Settings(톱니바퀴)** → **API** 클릭.
2. 두 가지 값을 복사합니다:
   - **Project URL** (예: `https://abcd1234.supabase.co`)
   - **Project API keys → anon public** 키
3. 이 폴더의 **`js/config.js`** 파일을 열어, 두 값을 아래처럼 붙여넣고 저장합니다:

   ```js
   window.KRS_CONFIG = {
     SUPABASE_URL: "https://여기에-프로젝트-URL.supabase.co",
     SUPABASE_ANON_KEY: "여기에-anon-public-키",
   };
   ```

   > 💡 anon key 는 웹에 공개되어도 안전합니다. (보안 정책으로 데이터가 보호됩니다.)

## 5단계 — 홈페이지 띄우기

이 사이트는 사진 업로드·로그인을 위해 **웹 주소(http)** 로 열어야 합니다.
파일을 더블클릭(`file://`)하면 일부 기능이 막히니, 아래 중 한 가지로 실행하세요.

- **가장 쉬움 — VS Code 사용 시**: 확장프로그램 *Live Server* 설치 →
  `index.html` 우클릭 → **Open with Live Server**.
- **파이썬이 있다면**: 이 폴더에서 명령창을 열고
  ```
  python -m http.server 5500
  ```
  실행 → 브라우저에서 `http://localhost:5500` 접속.

## ✅ 잘 됐는지 확인

1. 상단 메뉴 **회원 스토리** → **로그인** → **회원가입** 으로 계정 만들기.
2. 로그인 후 **✏️ 스토리 올리기** → 제목·내용·사진 입력 → 올리기.
3. 카드가 바로 보이면 성공! 다른 브라우저(또는 휴대폰)로 같은 주소에 접속하면
   **다른 사람에게도 똑같이 보입니다.** (클라우드 공유)

## 회원이 스토리 올리는 방법 (안내해 주세요)

1. 홈페이지 상단 **로그인** → 처음이면 **회원가입**(닉네임·이메일·비밀번호).
2. **회원 스토리** 메뉴 → **✏️ 스토리 올리기**.
3. 제목, 카테고리(자유글·가꾸기 후기·사진 자랑·질문답변·정원 자랑), 내용을 쓰고
   사진을 선택한 뒤 **스토리 올리기**.
4. 본인이 쓴 글은 카드를 눌러 열면 **삭제**할 수 있습니다.

## 🔑 회원 등급(권한) 설정 — 비공개 영역 관리

홈페이지에는 **일반 회원은 볼 수 없는** 비공개 영역이 있습니다.

| 영역 | 접근 가능 등급 |
|------|----------------|
| **회원관리**(명부·회비·입회일) | 집행위원 · 관리자 |
| **운영위원회**(분과 위원장 게시판) | 운영위원 · 집행위원 · 관리자 |
| **집행위원회**(회장·부회장·재무·회계·간사·고문) | 집행위원 · 관리자 |

등급은 4단계입니다: `member`(일반) · `operating`(운영위원) · `executive`(집행위원) · `admin`(관리자).
가입하면 누구나 처음에는 **일반회원(member)** 입니다.

### 등급 바꾸는 방법 (관리자)
1. Supabase 왼쪽 메뉴 **Table Editor** → **profiles** 테이블 선택.
2. 해당 회원의 행에서 **role** 값을 `operating` / `executive` / `admin` 중 하나로 바꾸고 저장.
3. **가장 먼저 본인 계정의 role 을 `admin` 으로 지정**하세요. 그래야 회원관리 등에 접근할 수 있습니다.
4. 해당 회원이 로그아웃 후 다시 로그인하면, 상단에 **"회원 전용 ▾"** 메뉴가 나타납니다.

> 🔒 권한은 화면뿐 아니라 **데이터베이스(RLS) 차원에서도 차단**되므로, 주소를 직접 입력해도 권한이 없으면 데이터가 보이지 않습니다.

## 💬 Q&A 자동응답

상단 **Q&A** 메뉴는 가입·회비·모임·장미 관리 등 자주 묻는 질문에 자동으로 답합니다.
별도 설정 없이 바로 동작하며, 질문/답변을 추가·수정하려면 **`data/faq.json`** 파일을 편집하면 됩니다.
(각 항목의 `keywords` 에 사용자가 입력할 만한 단어를 넣을수록 더 잘 찾아줍니다.)

## 📢 네이버 카페 최신글 (홈 노출)

홈에 "네이버 카페 최신글" 섹션이 있습니다. 네이버 카페는 로그인·보안(CORS) 때문에 웹사이트가
글을 **직접 자동으로 가져오기 어렵습니다.** 그래서 두 가지 방법을 제공합니다.

**방법 1) 직접 목록 관리 (간단·확실)**
`data/cafe.json` 파일을 열어 최신글을 적어주세요. 적은 그대로 홈에 표시됩니다.
```json
{ "title": "글 제목", "link": "https://cafe.naver.com/korearose", "date": "2026-05-01" }
```

**방법 2) RSS 자동 수집 (공개 게시판만)**
카페 게시판이 **공개(RSS 제공)**라면, 그 RSS 주소를 `js/config.js` 의 `CAFE_RSS_URL` 에 넣으세요.
홈이 열릴 때 최신글을 자동으로 불러옵니다. (회원 전용 비공개 게시판은 RSS 가 없어 불가)

> 💡 **완전 자동 동기화**(비공개 카페 포함, 정기적으로 자동 갱신)가 필요하면, 서버리스 함수로
> 카페 글을 주기적으로 수집해 사이트에 반영하는 방식이 필요합니다. 개발자에게 요청하세요.

## ❓ 도움말

- 화면에 "⚙️ 스토리 기능 설정이 필요합니다" 가 보이면 → `js/config.js` 값이 비어 있는 것입니다(4단계).
- 가입은 됐는데 로그인이 안 되면 → 3단계(이메일 인증 끄기)를 확인하세요.
- "회원 전용" 메뉴가 안 보이면 → profiles 테이블에서 role 을 올린 뒤 다시 로그인했는지 확인하세요.
- 더 다양한 기능(굿즈 결제 등)이 필요하면 개발자에게 요청하세요.
